概要

集計ツール(以降、当ツールと呼びます)は、SQL抽出支援ツール(以降、抽出ツールと呼びます)の実行結果に対して重複などの不要な項目の排除、ファイル形式の変換を行い、その結果を集計します。

当ツールは、抽出ツールパッケージに含まれます。

概念図

実行形式

当ツールは、コマンドラインで実行する単体アプリケーションです。キャラクタ・ユーザ・インタフェース(CUI)上で操作を行います。

入力ファイル

当ツールの入力について説明します。

  • 当ツールの入力は、解析対象ファイルへのパスや解析モードなどを記述した制御ファイルです。
    • 詳細は「コマンドライン仕様」の「入力」を参照ください。

出力ファイル

当ツールの出力について説明します。

  • 当ツールの出力は、CSVファイル(以降、集計結果ファイルと呼びます)およびERRファイル(以降、参照用ファイルと呼びます)です。
  • 集計結果ファイルは、報告書の作成などに便利です。参照用ファイルは、"viコマンド"を利用した対象ソースファイルの編集に便利です。

インストール

当ツールのインストールは、抽出対象ファイルへのアクセスが可能なマシンで実行します。また、インストール作業は、任意の OS ユーザで実行できますが、インストールを実行した OS ユーザ以外の OS ユーザが当ツールを実行する場合は、適切にアクセス権を設定する必要があります。

インストールディレクトリ作成

インストールディレクトリを作成し、当ツールのアーカイブファイルを作成したディレクトリにコピーします。なお、以降の説明では、当ツールのアーカイブファイルを『db_syntax_diff.tar.gz』、インストールディレクトリとして『/usr/local/db_syntax_diff』を作成し作業を行った場合を示します。 

$ mkdir /usr/local/db_syntax_diff
$ cp /media/disk/db_syntax_diff.tar.gz /usr/local/db_syntax_diff

アーカイブ展開

当ツールのアーカイブファイルを tar コマンドで展開します。

$ cd /usr/local/db_syntax_diff
$ tar xvzf db_syntax_diff.tar.gz

アーカイブ展開後のディレクトリ構成を以下に示します。 以下に含まれないファイル/ディレクトリは、抽出ツールのファイル/ディレクトリです。

[./]
|
+--- src
| |
| +--- db_syntax_diff_wrapper.sh ....... 実行スクリプト
| +--- csvtool_utf8.pl          ........ 当ツール本体
|
+--- config
| |
| +--- wrapper.ctl.sample ......... 制御ファイルのサンプル
|
+--- output               ......... 集計結果の出力格納ディレクトリ
+--- tmp                  ......... 一時ファイル格納ディレクトリ

アーカイブ展開後の『./src/db_syntax_diff_wrapper.sh』、『./src/convert_utf8.pl』、『./src/db_syntax_diff.pl』に実行権限が付与されていない場合は、実行権限を手動で付与してください。

$ chmod a+x ./src/db_syntax_diff_wrapper.sh
$ chmod a+x ./src/convert_utf8.pl
$ chmod a+x ./src/db_syntax_diff.pl

抽出ツールのインストールにある『makeの実行』、『環境変数の設定』、『動作確認』を行い、抽出ツールの実行環境を構築します。

本ツールの動作確認

本ツールの動作確認例を示します。 以下のように、Usage が出力された場合は正常にインストールが完了しています。

$ ./src/db_syntax_diff_wrapper.sh
db_syntax_diff_wrapper.sh <controlfile_path>

アンインストール

インストールディレクトリの削除

インストールディレクトリ全体を削除します。この作業でアンインストールは完了します。以下の例では、インストールディレクトリとして『/usr/local/db_syntax_diff』を作成して作業を行った場合を示します。

$ rm -rf /usr/local/db_syntax_diff

コマンドライン仕様

当ツールのコマンドライン仕様を以下に示します。

形式

引数として、制御ファイル(CONTROLFILE_PATH)を指定します。 
./src/db_syntax_diff_wrapper.sh CONTROLFILE_PATH

入力

CONTROLFILE_PATH
  • 制御ファイルへのパス(絶対パス、相対パスどちらも可)を指定します。
    • 当ツールのパッケージにはwrapper.ctl.sampleが含まれていますが、ファイル名に規定はありません。 任意のファイル名を付けて管理してください。
  • 制御ファイルには、抽出対象ファイルへのパスや解析するモード等を記述します。
    • 具体的な設定項目は、以下の項目です。
      設定名 説明 補足
      TARGET_PATH 抽出対象ファイルのディレクトリへのフルパス (入力必須項目)
      TARGET_EXT 抽出対象ファイルの拡張子 設定値を与えない場合、「TARGET_PATH」で指定したディレクトリ配下のファイル全てを入力対象として読み込みます。
      TARGET_MODE 抽出ツールの実行モード "java", "c", "sql", "cpp"のいずれかを小文字で記述します。
      (入力必須項目)
      TARGET_CHAR 抽出対象ファイルのエンコード "utf8", "eucjp", "shiftjis"のいずれかを小文字で記述します。
      (入力必須項目)
      POSTGRESQL_VERSION 移行先環境でのPostgreSQLバージョン メジャーバージョン(上位2桁)を記述します。(例：8.4)
      (default:なし)
      ORAFCE_VERSION 移行先環境でのorafceバージョン フルバージョン(3桁)を記述します。(例：3.0.0)
      (default:なし)
      OUTPUT_DIR_NAME 出力ディレクトリ名 抽出結果と実行ログを出力するディレクトリを記述します。
      (入力必須項目)
      STDERR_OUTPUT_FLAG 実行ログの出力モード
      ※実行ログとは抽出ツールから出力されるログを指す。      		 実行ログの出力モードは下記のいずれかを小文字で記述します。
      「both」(標準エラー出力+ファイル出力)
      「screen」(標準エラー出力のみ)
      「file」(ファイル出力のみ)
      「空文字」(実行ログを出力しない)
      (default:なし)
      STDERR_FILE_NAME 実行ログの出力ファイル名 実行ログの出力ファイル名を記述します。
      (STDERR_OUTPUT_FLAGが「both」or「file」の場合のみ、入力必須項目)
      (default:なし)
      INCLUDE_DIRECTORY_NAME ヘッダファイルのディレクトリへのパス ヘッダファイルの検索先を記述します。
      (default:なし)
  • 各設定項目は「設定名="設定値"」の形式で記述します。
  • 設定名より前の空白・タブは許容します。
  • ＝の前後の空白は許容しません。
  • 同一設定の複数記述時は後から記述した設定が有効になります。
  • 設定名以外を設定した場合でも、エラー終了はしません。
  • 1行に設定値は1つだけ記述します。
  • 行の先頭が「#」から始まる場合は、その行をコメントとして扱います。
  • 設定値の記述形式はシェルに準拠します。
  • 制御ファイルのエンコーディングは、OSのエンコーディングと合わせる必要があります。

    • 戻り値

    0 正常終了
    入力ファイルに対する解析、および観点に対する報告が正常に完了したことを示します。
    1 異常終了
    当ツール実行時に何らかの異常が発生したことを示します。

    利用方法

    当ツールの利用方法として、当ツールの実行手順および出力結果の確認方法について記述します。 本章では、当ツールのパッケージは「/usr/local/db_syntax_diff」にインストールされているものとして記述します。

    当ツールの実行手順

    以下の手順に従って当ツールを実行します。
    1. 抽出対象ファイルの用意
    2. 制御ファイルの作成
    3. 当ツールの実行
    4. 出力結果の確認
    以下に、Javaのソースファイルを解析する例を示します。
    1. 抽出対象ファイルを用意します
      2. ここの例では、「/tmp/test」ディレクトリに、以下の2ファイルを用意したとして例を示します。
      ファイル名ファイル内容
      test01.java 
      import java.sql.*;
import java.util.*;

public class test01 {
    String sqlString = "SELECT * FROM dual";

    public ResultSet testMethod() throws SQLException {
        ResultSet rs = stmt.executeQuery(sqlString);
        ResultSet rs2= stmt.executeQuery("SELECT * FROM dual");
        return rs;
    }
}
      test02.java 
      import java.sql.*;
import java.util.*;

public class test02 {
    String sqlString = "DELETE mytbl";

    public ResultSet testMethod() throws SQLException {
        ResultSet rs = stmt.execute(sqlString);
        ResultSet rs2= stmt.executeQuery("SELECT sysdate FROM dual");
        return rs;
    }
}

    2. 制御ファイルを作成します
      3. 上記のソースファイルを解析するように制御ファイルを作成します。 ここでは、configディレクトリ配下にあるwrapper.ctl.sampleを/tmpディレクトリに wrapper.ctlという名前でコピーし、修正した例を示します。 
      $ cp /usr/local/db_syntax_diff/config/wrapper.ctl.sample /tmp/wrapper.ctl
$ vi /tmp/wrapper.ctl
... 編集 ...
$ cat /tmp/wrapper.ctl
TARGET_PATH="/tmp/test"
TARGET_EXT="java"
TARGET_MODE="java"
TARGET_CHAR="utf8"
POSTGRESQL_VERSION=
ORAFCE_VERSION=
OUTPUT_DIR_NAME="java_test"
STDERR_OUTPUT_FLAG=""
STDERR_FILE_NAME=""
INCLUDE_DIRECTORY_NAME=""

    3. 当ツールを実行します
      4. 上記で作成した制御ファイルを指定して、当ツールを実行します。 
      $ ./src/db_syntax_diff_wrapper.sh /tmp/wrapper.ctl

    4. 出力結果を確認します
      5. 当ツールは、outputディレクトリ配下に制御ファイルのOUTPUT_DIR_NAMEで指定した ディレクトリ(上記の例ではjava_testディレクトリ)を作成し、そのディレクトリ配下に csvディレクトリとeditdataディレクトリを作成します。 当ツールが標準出力にメッセージを出力しない場合、csvディレクトリに複数の集計結果ファイル、 editdataディレクトリに複数の参照用ファイルを出力します。それぞれ利用方法を次節で説明します。
      標準出力に「修正箇所はありませんでした。」というメッセージを出力した場合、 非互換箇所がないことをあらわしています。この場合、当ツールはcsvディレクトリに1つの 集計結果ファイルのみ出力します(後述するresult.csvのみ出力します)。

    出力結果の利用方法

    非互換箇所が検出された場合の出力結果である集計結果ファイルおよび参照用ファイルの利用方法について説明します。
    集計結果ファイルは主にシステムに含まれる非互換箇所の集計等を行う目的で利用します。
    一方、参照用ファイルは主に"viコマンド"等のエディタを用いて、抽出対象ファイルを編集する目的で利用します。

    集計結果ファイルの利用方法

    当ツールは、OUTPUT_DIR_NAMEで指定したディレクトリ(上記の例ではjava_testディレクトリ)内のcsvディレクトリに、 下記4つの集計結果ファイルを出力します。
    主に、result_id_sjis.csv、result_id.csvを確認し、非互換箇所の集計を行います。 result.csvは、「クラス名」や「メソッド名」などを確認したい場合のみ利用してください。 result_score.csvは、修正に掛かる工数の見積もりを行う際に利用してください。 SJISで出力されるものはWindows上で確認し、SJIS以外(TARGET_CHARで指定した文字コード)で出力されるものは Linux上で確認することを想定しています。
    • result_id_sjis.csv ... 同じ非互換項目(メッセージIDが同一のもの)毎にまとめ、総数を出力します(文字コードはSJIS)
    • result_id.csv ... result_id_sjis.csvと同じ内容です(文字コードはTARGET_CHARで指定)
      • (メッセージID),(タイプ),(エラーレベル),(メッセージ),(修正数)
    • result.csv ... 1つの非互換項目を1行づつ出力します(文字コードはSJIS)
      • (フルパス),(クラス名),(メソッド名),(行数),(カラム位置),(メッセージID),(タイプ),(エラーレベル),(スコア),(メッセージ),(ターゲット文字列)
    • result_score.csv ... 1行毎のスコア合計値と対象文字列を出力します(文字コードはTARGET_CHARで指定)
      • (行毎のスコア合計値),(ターゲット文字列),(フルパス),(行数)

      「エラーレベル」は、修正の難易度を以下のレベルであらわしています。
      エラーレベル 説明
      LOW1 機械的に直せるもの
      LOW2 機械的修正は出来ないが修正が容易なもの
      MIDDLE 機械的修正が出来ず、周辺処理を確認しながら修正が必要なもの
      HIGH 機械的修正が出来ず、PostgreSQLに代替機能がないために検討が必要なもの
      CHECK_LOW1
      CHECK_LOW2
      CHECK_MIDDLE
      		 目視確認にて、移植時の修正有無、難易度を確認する必要があるもの。
      "CHECK_"につづく難易度(LOW1,LOW2,MIDDLE,HIGHのいずれか)は、参考情報として可能性のある難易度を示しています
      ERROR LV1,2,3,4,5
      WARNING
      FATAL      		 目視確認にて、移植時の修正有無、難易度を確認し、対応策を検討する必要があるもの

    以下、result_id_sjis.csv、result_id.csvを用いた集計の一例です。
    前出の例の出力結果であるresult_id.csvを元に説明します。 
      FNC-086-001,FUNC,LOW2,"SYSDATE関数は未サポートです。",1
SQL-107-008,SQL,LOW2,"FROMの省略は未サポートです。",1
SQL-119-706,SQL,LOW1,"DUAL表が参照されています。DUAL表は未サポートです。PostgreSQLはFROM句を記述しない形式をサポートしていますので、SQLを検討してください。",3
    • 非互換項目毎の非互換項目数を数える
      • 各行の「修正数」列が、非互換項目毎の非互換項目数を表しています。
      それぞれの行の「修正数」列の1、1、3が、各非互換項目の項目数になります。
    • 全非互換項目数を数える
      • 全ての行の「修正数」列の合計を求めることで、全非互換項目数を数えることができます。
      全ての行の「修正数」列の合計である5(=1+1+3)が、全非互換項目数になります。
    • エラーレベル毎の非互換項目数を数える
      • 「エラーレベル」列でフィルタリングを行った上で、「修正数」列の合計を求めることで、 エラーレベル毎の非互換項目数を数えることができます。
      例えば、「エラーレベル」が "LOW2" の項目でフィルタリングして、「修正数」列を 合計した2(=1+1)が、"LOW2"の非互換項目数になります。

    参照用ファイルの利用方法

    当ツールは、OUTPUT_DIR_NAMEで指定したディレクトリ(上記の例ではjava_testディレクトリ)内のeditdataディレクトリに、 非互換項目分の参照用ファイルを出力します。ファイル名は"result_[メッセージID].err"となります。 出力ファイルのフォーマットは以下の通りです。エディタの機能を用いて該当の抽出対象ファイルの修正箇所の 目視や編集が容易に行えます。 
    (フルパス):(行数):(カラム位置):(メッセージ)
    以下、viコマンドを用いて参照用ファイルを利用する例を示します。
    前節でdual表を参照している非互換箇所(メッセージID＝SQL-119-706)を目視、編集する例です。 
    $ vi -c cw -q output/java_test/editdata/result_SQL-119-706.err
    viコマンドを、オプション"-c cw -q"を指定して実行します。

    以下のような画面になります。


    上記のように、上半分(以下、ソースウィンドウと呼びます)に対象のソースが表示され、 下半分に参照用ファイルの内容(以下、リストウィンドウと呼びます)が一覧されます。 ソースウィンドウで対象のソースファイルの確認・編集を行ってください。
    「Ctrl-w」を押したあと「w」を押すと、ソースウィンドウとリストウィンドウ間で カーソルを移すことができます。 リストウィンドウでカーソルを上下し「Enter」を押すことで、他の非互換箇所を 表示することができます(表示するとカーソルはソースウィンドウの該当行に移ります)。

    ソースウィンドウに表示されたソースは編集可能です。 編集を行った後、viコマンドの変更反映手順(":w")でファイルを更新できます。 確認・編集が完了したら、ソースウィンドウ、リストウィンドウの両方をviコマンドの 終了手順(":q")で終了させます。

    ～便利な利用方法～

    当ツールで扱える抽出対象ファイルは、Java、JSP、Pro*C、SQLファイルです。 このため、XMLに記述されたSQLやログに出力されたSQLの解析には利用できません。
    しかし、XMLやログからSQLを抽出し、SQLファイルを作成すればSQLファイルとして扱うことができます。 参照用ファイルは作成したSQLファイルの箇所を指し示すため、抽出対象ファイルを直接編集することはできませんが、 システムに含まれる非互換箇所を確認することが可能になります。
